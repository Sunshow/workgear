# Proposal: 流程管理页面添加返回看板导航

## 为什么做（Why）

当前用户从看板页面进入流程管理页面后，页面上没有提供任何返回看板的入口。用户只能通过以下方式返回：

1. 浏览器后退按钮
2. 左侧边栏点击"项目" → 项目列表 → 点击项目卡片 → 回到看板

这严重影响了用户在看板与流程管理之间的导航效率，打断了工作流。

### 用户痛点

- 流程管理页面是从看板页面的"流程管理"按钮进入的，用户心智模型中期望有对称的返回路径
- 工作流编辑器页面已有返回按钮（返回流程列表），但流程列表页面本身缺少返回看板的按钮，导航链路断裂
- 依赖浏览器后退按钮不可靠（用户可能从其他路径进入）

## 做什么（What）

在流程管理页面（`/projects/:projectId/workflows`）的页面头部添加一个"返回看板"按钮，点击后导航回 `/projects/:projectId/kanban`。

### 具体方案

- 在流程管理页面 header 左侧添加 ArrowLeft 图标按钮
- 复用工作流编辑器中已有的返回按钮交互模式（`ghost` variant + ArrowLeft icon）
- 点击后使用 `navigate()` 跳转到看板页面

## 影响范围（Scope）

### 涉及模块

| 模块 | 影响 | 说明 |
|------|------|------|
| kanban | MODIFIED | 更新 spec，补充导航场景描述 |
| web/pages/workflows | 代码变更 | 添加返回按钮到页面 header |

### 不涉及

- 后端 API 无变更
- 路由配置无变更（路由已存在）
- 无新增组件（复用现有 Button + ArrowLeft）
- 无状态管理变更

### 风险评估

- **风险等级：低** — 纯前端 UI 变更，不涉及数据流或业务逻辑
- 复用已有设计模式，无技术风险
